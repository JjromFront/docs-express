Express.js

Introduccion

Express.js es un framework web para Node.js que simplifica la creación de aplicaciones web y APIs. Destaca por su simplicidad, enrutamiento flexible, uso de middleware y soporte para renderizado de vistas. Express.js tiene una gran comunidad y ofrece un ecosistema rico en módulos y herramientas. Es ideal para construir aplicaciones web rápidas y escalables en Node.js.

Configuracion

Para empezar, debes asegurarte de tener Node.js instalado en tu máquina. Puedes verificarlo ejecutando el siguiente comando en tu terminal:

node v

Si Node.js no está instalado, visita el sitio web oficial de Node.js (https://nodejs.org) y sigue las instrucciones de instalación adecuadas para tu sistema operativo.

Una vez que tengas Node.js instalado, puedes crear un nuevo proyecto de Express.js. Abre tu terminal y ejecuta los siguientes comandos:

mkdir mi-proyecto
cd mi-proyecto
npm init -y

Esto creará una nueva carpeta para tu proyecto y generará un archivo package.json con la configuración predeterminada, a continuación, instala Express.js ejecutando el siguiente comando, el cual descargara express en tu proyecto:

npm i express

Bien hecho ahora tenemos ya tenemos un proyecto de express, sigamos con la siguiente seccion.

Creación de una aplicación básica

Ahora que tienes Express.js configurado, puedes crear una aplicación básica. Crea un archivo llamado app.js (o cualquier otro nombre que prefieras) en la raíz de tu proyecto y ábrelo con tu editor de código.

En el archivo app.js, escribe el siguiente código para crear una aplicación Express básica: 

                        
    const express = require("express")
    const app = express();

    app.get('/', (req, res) =>  {
        res.send('¡Hola, mundo!');
    });

    app.listen(3000, () => {
        console.log('La aplicación está escuchando en el puerto 3000');
    });

En este código, importamos Express.js y creamos una nueva instancia de la aplicación Express llamada app. Luego, definimos una ruta raíz ('/') utilizando el método get(). Cuando se acceda a esa ruta mediante una solicitud GET, enviaremos la respuesta '¡Hola, mundo!' al cliente.

Finalmente, llamamos al método listen() para iniciar el servidor en el puerto 3000. El método listen() toma una función de devolución de llamada opcional que se ejecuta una vez que el servidor comienza a escuchar las solicitudes entrantes.

Para ejecutar la aplicación, vuelve a tu terminal y ejecuta el siguiente comando en la raíz de tu proyecto:

node app.js

Esto iniciará el servidor y podrás acceder a tu aplicación abriendo un navegador y visitando http://localhost:3000. Verás el maravilloso mensaje '¡Hola, mundo!' en la página.

Enrutamiento

Express.js facilita la creación de rutas para manejar diferentes solicitudes HTTP. Puedes definir rutas para métodos como GET, POST, PUT y DELETE. Aquí hay un ejemplo de cómo definir diferentes rutas:

                        
    app.get('/', (req, res) => {
        res.send('Ruta raíz');
    });

    app.post('/users', (req, res) => {
        res.send('Crear un nuevo usuario');
    });

    app.put('/users/:id', (req, res) => {
        const userId = req.params.id;
        res.send(`Actualizar el usuario con ID ${userId}`);
    });

    app.delete('/users/:id', (req, res) => {
        const userId = req.params.id;
        res.send(`Eliminar el usuario con ID ${userId}`);
    });

En este ejemplo, hemos definido cuatro rutas diferentes:

  1. '/' es la ruta raíz y responde a las solicitudes GET con el mensaje 'Ruta raíz'

  2. '/users' es una ruta para crear nuevos usuarios y responde a las solicitudes POST con el mensaje 'Crear un nuevo usuario'

  3. '/users/:id' es una ruta para actualizar o eliminar usuarios existentes. Utilizamos :id como un marcador de posición para el ID del usuario y accedemos a él utilizando req.params.id.

  4. La ruta '/users/:id' responde a las solicitudes PUT con un mensaje que indica la intención de actualizar el usuario con el ID especificado.

  5. La ruta '/users/:id' responde a las solicitudes DELETE con un mensaje que indica la intención de eliminar el usuario con el ID especificado.

Esto es solo un ejemplo básico de enrutamiento. Express.js proporciona muchas más funcionalidades para manejar rutas, como enrutamiento modular, enrutamiento basado en controladores y enrutamiento con expresiones regulares.

Manejo de solicitudes y respuestas

Cuando se realiza una solicitud a una ruta específica, puedes utilizar req y res (objetos de solicitud y respuesta, respectivamente) para manipular y enviar datos de vuelta al cliente. A continuación, se muestra un ejemplo de cómo manejar una solicitud GET y enviar una respuesta JSON:

                        
    app.get('/users', (req, res) => {
        const users = [
        { id: 1, name: 'Usuario 1' },
        { id: 2, name: 'Usuario 2' },
        { id: 3, name: 'Usuario 3' }
        ];
                              
        res.json(users);
    });

En este ejemplo, hemos definido una ruta '/users' que responde a las solicitudes GET con un arreglo de usuarios en formato JSON. Utilizamos el método res.json() para enviar la respuesta en formato JSON al cliente.

Express.js proporciona muchos otros métodos en el objeto res para enviar respuestas con diferentes formatos, como res.send() res.render(), res.redirect(), entre otros.

Middleware

Una de las características más poderosas de Express.js es su capacidad para trabajar con middleware. El middleware es una función que se ejecuta antes de que se maneje una solicitud. Puede realizar tareas como el registro de solicitudes, el análisis de datos de entrada, la autenticación y la autorización, entre otros. Express.js utiliza el concepto de middleware para proporcionar una arquitectura modular y extensible.

Aquí hay un ejemplo de cómo utilizar middleware en Express.js: 

                        
    // Middleware de registro de solicitudes
    app.use((req, res, next) => {
        console.log(`[${new Date().toISOString()}] ${req.method} ${req.url}`);
        next();
    });
                            
    // Middleware de análisis de datos de entrada
    app.use(express.json());
                            
    // Middleware de autenticación
    app.use((req, res, next) => {
        // Realizar lógica de autenticación
        if (req.isAuthenticated()) {
        next(); // Permitir acceso a rutas protegidas
        } else {
            res.status(401).send('No autorizado');
    }
                            });

En este ejemplo, hemos definido tres middleware diferentes:

  1. El primer middleware registra cada solicitud mostrando la fecha y hora, el método HTTP y la URL de la solicitud.

  2. El segundo middleware utiliza express.json() para analizar los datos de entrada en formato JSON antes de que se manejen las solicitudes.

  3. El tercer middleware realiza una lógica de autenticación básica. Si el usuario está autenticado, permite el acceso a las rutas protegidas llamando a next(). De lo contrario, envía una respuesta con el código de estado 401 (No autorizado).

Express.js admite middleware de terceros y puedes encontrar una amplia gama de módulos de middleware en el registro de paquetes de npm (https://www.npmjs.com) para agregar funcionalidades adicionales a tu aplicación.

Plantillas de vista

Express.js también admite el renderizado de vistas utilizando plantillas. Puedes utilizar diferentes motores de plantillas como EJS, Pug (anteriormente conocido como Jade), Handlebars, entre otros. Aquí hay un ejemplo utilizando el motor de plantillas EJS:

Primero, instala el paquete ejs ejecutando el siguiente comando:

npm install ejs

Luego, configura Express.js para utilizar EJS como motor de plantillas en tu archivo app.js: 

                        
    app.set('view engine', 'ejs');
    app.set('views', path.join(__dirname, 'views'));

En este ejemplo, estamos utilizando app.set() para establecer 'view engine' en 'ejs' lo que indica que usaremos EJS como motor de plantillas. Además, estamos estableciendo 'views' para que Express.js busque las plantillas en una carpeta llamada 'views' en la raíz de tu proyecto.

Finalmente, puedes crear una plantilla EJS en la carpeta 'views' y renderizarla utilizando Express.js. Por ejemplo, crea un archivo llamado index.ejs en la carpeta 'views' con el siguiente contenido:

                        
    <!DOCTYPE html>
    <html>
    <head>
        <title>¡Hola, mundo!</title>
    </head>
    <body>
        <h1>¡Hola, <%= nombre %>!</h1>
    </body>
    </html>

Luego, actualiza tu archivo app.js para manejar la solicitud a la ruta raíz y renderizar la plantilla:

                        
    app.get('/', (req, res) => {
        const nombre = 'usuario';
        res.render('index', { nombre });
    });

En este ejemplo, estamos utilizando res.render() para renderizar la plantilla 'index.ejs' y pasarle un objeto que contiene la variable nombre

Cuando se accede a la ruta raíz, Express.js renderizará la plantilla utilizando EJS y enviará la página HTML resultante al cliente, reemplazando <%= nombre %> con el valor de nombre proporcionado.

Esta es solo una introducción básica al uso de plantillas de vista con Express.js. Puedes explorar la documentación de los motores de plantillas específicos para obtener más información sobre cómo utilizarlos y aprovechar al máximo sus características.
